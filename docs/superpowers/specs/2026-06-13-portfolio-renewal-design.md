## 1. コンセプト

「最低条件を並べた名刺サイト」から「課題を解決できる人だと伝わる提案サイト」へ。

上品で信頼できる土台に、クリエイティブな感度を効かせる。**デザインから実装までを一人で貫ける**ことと、**作り手の人柄**を、直クライアント（事業会社・個人事業主）に伝えることが目的。

### ターゲットと役割
- 想定読者: 直クライアント（事業会社・個人事業主）。発注前に「信頼できるか」「課題を解決してくれるか」を見極めたい層。
- 役割の見せ方: デザイン〜実装の両方を担えることを軸に据える。

### 旧サイトからの改善方針
1. 抽象的な3強み（見やすいデザイン/スマホ対応/丁寧なコーディング）→ 提供価値の言葉へ磨き直す。
2. スキルが HTML/CSS/JS のみ → Design 領域（設計・UI・Figma）を前面に出し「両方できる」を可視化。
3. 作品が一覧止まり → ケーススタディ型で「課題→解決→成果」を深掘り。
4. 自分視点のキャッチ → 相手目線のメッセージへ。
5. 人柄の不在 → About で人となり・価値観・なぜこの仕事かを語る。

## 2. トンマナ

誠実・丁寧・信頼 × クリエイティブな感度。
静の上品を土台にし、見せ場だけ演出を効かせるバランス型。過剰演出はしない。

## 8. 実装ガードレール（UX / A11y / Performance）

ui-ux-pro-max の design-system 検証と UX ガイドラインに基づく。構成（Portfolio Grid: ビジュアル先行 → グリッド → About → Contact）は推奨と一致。以下は実装時の必須チェック。

### アニメーション（"Motion-Driven" の注意点）
- **prefers-reduced-motion: reduce を尊重**（High）。GSAP / ScrollTrigger は当該メディアクエリで無効化 or 大幅に弱める。スクロールジャック・強制パララックスは避ける。
- 1ビューで動かすのは主役1〜2要素まで（過剰演出禁止）。
- enter は ease-out、exit は ease-in。linear は使わない。
- アニメは `transform` / `opacity` のみ。width/height/top/left は animate しない（リフロー回避）。

### パフォーマンス
- 作品サムネ等は WebP/AVIF＋`loading="lazy"`（fold 下）。Hero 直近のみ eager。（既存 `/optimize-images` 活用）
- 画像は width/height もしくは aspect-ratio を指定し CLS を防ぐ。
- フォントは `font-display: swap`。Fraunces / Shippori Mincho の必要ウェイトのみ読み込む。

### アクセシビリティ / インタラクション
- コントラスト 4.5:1 を確認：Ink#17120e / Paper#f7f5f0、Green#2f4a3a 上の白文字、Paper 上の Green 文字（いずれも基準クリア想定だが実装時に検証）。
- クリッカブル要素に `cursor: pointer`、ホバーは 150–300ms のスムーズな遷移。
- キーボード操作のフォーカスリングを残す（消さない）。見出しは h1→h6 の順序を守る。
- 色だけで情報を伝えない（リンク/状態にテキストや下線を併用）。

### レスポンシブ
- ブレークポイントは既存トークン基準（375 / 768 / 1024 / 1440 目安）。モバイルファースト。
- メタ数値（2022–2026 / 05 projects 等）は等幅数字（tabular figures）で桁ズレ防止。
